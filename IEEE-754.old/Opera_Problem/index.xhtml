IEEE-754 Analyzer

Input Format
Syntax Entered: none

Normalized decimal value: 0

Normalized binary value: 0

Instructions

You must have JavaScript enabled to do anything with this web page.

This page lets you examine the relationships among binary and decimal numbers and three number formats described by the IEEE-754-2008 floating-point standard. You can enter a numeric value in any one of five formats, and see all five corresponding values, along with an analyses of the binary structure of the floating-point formats.

You can create multiple analyzers so you can compare multiple values at the same time.

Input Formats

  • Auto

    If the auto option is checked, you can enter a value in any of the formats listed below. If you enter a valid decimal number, it will be treated as such even though it might also be a valid binary or hexadecimal number. For example, 10101010 could be the decimal number 10,101,010; the the binary number 170; or it could be a 32-bit hexadecimal value. In ambiguous cases like this, use the radio buttons to force the value to be interpreted using the format you want.

  • Decimal

    You can enter decimal numbers as whole numbers (123), decimal fractions (123.456), or mixed numbers (1 7/8). The number may be signed, and you can use exponential notation, even for the parts of mixed numbers. (1.23e2 or 1e3 1e2/7e1 for example; corresponding to 123 and 1000 100/70, repsectively)

  • Binary

    Binary numbers may be signed whole numbers or binary fractions with an optional exponent. Write the exponent in decimal, but it will be base 2. (101.1e-2 is the same as 1.011, for example.)

  • Hexadecimal

    This analyzer recognizes three IEEE-754-2008 standard floating-point formats: binary32, commonly called single-precision; binary64, commonly called double-precision; and binary128. Enter exactly 8, 16, or 32 hexadecimal digits to enter a value in one of these formats.
